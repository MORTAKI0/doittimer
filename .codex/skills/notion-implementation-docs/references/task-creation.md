# Task Creation

Create tasks only for work that still needs to happen, unless the user explicitly asks for historical backfill.

## Good task traits
- one clear action
- 1 to 2 days of work when possible
- measurable acceptance criteria
- linked to plan and implementation page
- clearly scoped

## Good task examples
- add integration tests for auth callback flow
- refactor dashboard fetch logic into server repository
- document environment variables and local setup
- harden validation for create-project form

## Avoid
- giant umbrella tasks
- tasks for already finished work unless requested
- vague names like "fix stuff" or "improve feature"
