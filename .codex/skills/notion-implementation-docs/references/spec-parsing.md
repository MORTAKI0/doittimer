# Spec Parsing

When a spec exists, extract these sections:

## 1. Core problem
- What user or business problem is being solved?
- What outcome matters?

## 2. Scope
- In scope
- Out of scope
- Explicit exclusions

## 3. Requirements
Extract:
- functional requirements
- non-functional requirements
- UX expectations
- technical constraints
- security/validation requirements
- acceptance criteria

## 4. Ambiguities
Capture unclear points such as:
- unspecified edge cases
- ownership assumptions
- missing data contracts
- unclear rollout expectations
- partial success behavior
- unclear auth rules

## 5. Priorities
Identify:
- must-have
- should-have
- follow-up
- deferred work

## Clarifications block format

Use this before implementation planning when the spec is incomplete:

## Clarifications / Assumptions
- Assumption:
- Ambiguity:
- Proposed interpretation:
- Risk if wrong:
