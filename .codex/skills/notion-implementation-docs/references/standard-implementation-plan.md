# Standard Implementation Plan

Use for larger, multi-part, or multi-phase work.

## Sections
1. Overview
2. Linked source/spec
3. Requirements summary
4. Current implementation status
5. Phase breakdown
6. Dependencies
7. Risks / open questions
8. Success criteria
9. Remaining work
10. Linked tasks
11. Progress checkpoints

## Phase guidance
Each phase should include:
- objective
- completed work
- remaining work
- risks
- exit criteria
