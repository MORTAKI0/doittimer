# Progress Tracking

Use progress tracking when work continues after the initial implementation doc.

## Cadence
Use updates:
- after meaningful code changes
- after phase completion
- when blockers appear
- when remaining work changes materially

## What to update
- implementation doc summary if scope changed
- plan phase status
- task statuses
- blockers and decisions
- next recommended step

## Status language
Prefer:
- not started
- in progress
- blocked
- done
- deferred
